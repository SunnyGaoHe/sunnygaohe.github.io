# Objctive-C基础教程 * Cocoa 和 Objective-C 是 iOS 和 OS X 的核心, OC 是 C 的超集, 融合了Smalltalk 语言的优势 (面向对象), 1985年, 史蒂夫.乔布斯创立了 NeXT 公司, NeXT公司选用 Unix 作为操作系统, 开发了 NextSTEP (OC 开发的一款用户界面工具包), 1996年, 乔布斯回归 Apple , NextSTEP 改为 Cocoa. * Hello World, 第一个 OC 程序: ```` #import int main (int argc, const char *argv[]) { NSLog(@"Hello World"); return 0; } ``` * Xcode 通过 `.m` 拓展名来表示文件使用的是 OC 代码, `.cpp` 表示使用的是 C++ 代码, `.c` 表示的是使用 C 代码, Xcode 使用 LLVM 编译器. * 使用 `#import " "` 和 使用 `#import < >` 的好处是无论使用该指令引用相同的头文件多少次 Xcode 只会引用一次. * `NS` 是 Cocoa 中所有内容的前缀, 如 `NSString` `NSArray` `NSMutableDictionary` 等. * `NSLog(@" ")` 用来打印内容, `%@` 可以打印字符串, 对象, C类型的数据. ``` NSLog(@"GaoHe is a %@", @"genius"); //打印字符串 NSLog(@"Tody is %@", [NSDate date]); //打印对象 int a = 10, b = 20; NSLog(@"%d + %d = %d", a, b, a + b); //打印 C 类型的数据 ``` * BOOL 类型提供 `YES` 和 `NO` 两个值, BOOL 类型实际上是一种带符号的字符类型 (signed char) 的类型定义 (typedef), 它使用8位的存储空间, 通过 `#define` 指令将 `YES` 定义为1 `NO` 定义为 0, 所以编译器将 BOOL 认作8位二进制数, 注意, 在逻辑判断中不能与 BOOL 中的 `YES` 进行判断, 因为 与 `YES` 比较是不安全的. ``` BOOL isPlaying; /** *一些逻辑代码 */ if (!isPlaying) { NSLog(@"Is playing now"); } else { NSLog(@"Is pause now"); } ```
